# Project Cleanup Guide

This document explains what files and directories can be safely removed from the project.

## Files/Directories That Should Be Removed

### Development Dependencies (Not Needed in Production)

1. **`node_modules/`** - Node.js dependencies
   - **Location**: `pixera_frontend/node_modules/`
   - **Why remove**: Large directory (~100MB+), only needed for building
   - **When needed**: Reinstalled automatically with `npm install`

2. **`venv/`** - Python virtual environments
   - **Locations**: 
     - `venv/` (root)
     - `pixera_backend/venv/`
   - **Why remove**: Large directory, platform-specific
   - **When needed**: Recreated automatically on production server with `start_production.sh`

### Build Artifacts

3. **`dist/`** - Frontend build output
   - **Location**: `pixera_frontend/dist/`
   - **Why remove**: Generated files, rebuilt when needed
   - **When needed**: Recreated with `npm run build`

4. **`production_build/`** - Production build directory
   - **Location**: Root directory
   - **Why remove**: Generated directory, rebuilt when needed
   - **When needed**: Recreated with `build_production.sh`

### Cache Files

5. **`__pycache__/`** - Python bytecode cache
   - **Locations**: Throughout the project
   - **Why remove**: Auto-generated, not needed
   - **When needed**: Automatically recreated by Python

6. **`*.pyc`, `*.pyo`** - Compiled Python files
   - **Why remove**: Auto-generated, not needed
   - **When needed**: Automatically recreated by Python

### Log Files

7. **`logs/*.log*`** - Application log files
   - **Location**: `logs/`
   - **Why remove**: Can grow large, not needed for development
   - **When needed**: Created automatically when application runs

8. **`logs/*.pid`** - Process ID files
   - **Location**: `logs/`
   - **Why remove**: Runtime files, not needed
   - **When needed**: Created automatically when application runs

### Old/Unused Files

9. **`pixera_frontend/server.js`** - Old Express relay server
   - **Why remove**: Not used with current production setup
   - **Replacement**: Frontend is served via `serve` or nginx

10. **`pixera_frontend/public/`** - Old build artifacts
    - **Why remove**: Using Vite build system instead
    - **Replacement**: `pixera_frontend/dist/` (generated by Vite)

11. **`start_pixera_countdowns.sh`** - Old development start script
    - **Why remove**: Replaced by `start_production.sh`
    - **Keep for**: Reference if needed

12. **`package-lock.json`** (root) - If exists without `package.json`
    - **Why remove**: Should only be in `pixera_frontend/`
    - **Keep**: `pixera_frontend/package-lock.json`

### Duplicate Config Files

13. **`postcss.config.cjs`** - If `postcss.config.js` exists
    - **Why remove**: Duplicate, keep `.js` version

14. **`tailwind.config.cjs`** - If `tailwind.config.js` exists
    - **Why remove**: Duplicate, keep `.js` version

## How to Clean Up

### Automatic Cleanup

Run the cleanup script:

```bash
./cleanup.sh
```

This will remove all unnecessary files and directories listed above.

### Manual Cleanup

If you prefer to clean up manually:

```bash
# Remove Python cache
find . -type d -name "__pycache__" -exec rm -rf {} +
find . -type f -name "*.pyc" -delete

# Remove virtual environments
rm -rf venv/ pixera_backend/venv/

# Remove node_modules
rm -rf pixera_frontend/node_modules/

# Remove build artifacts
rm -rf pixera_frontend/dist/ production_build/

# Remove old log files
rm -f logs/*.log* logs/*.pid

# Remove old/unused files
rm -f pixera_frontend/server.js
rm -rf pixera_frontend/public/
```

## What Gets Recreated Automatically

These will be recreated as needed:

- **`venv/`** - When you run `start_production.sh` or manually create it
- **`node_modules/`** - When you run `npm install` in `pixera_frontend/`
- **`dist/`** - When you run `npm run build` in `pixera_frontend/`
- **`production_build/`** - When you run `build_production.sh`
- **`__pycache__/`** - Automatically by Python when running code
- **`logs/*.log`** - When the application runs

## Before Deployment

Before building for production, it's recommended to run:

```bash
./cleanup.sh
./build_production.sh
```

This ensures a clean build without unnecessary files.

## File Sizes (Approximate)

- `node_modules/`: ~100-200 MB
- `venv/`: ~50-100 MB
- `__pycache__/`: ~1-5 MB
- `dist/`: ~1-5 MB (after build)
- `production_build/`: ~10-20 MB (after build)

**Total space saved**: ~150-300 MB

## Git Considerations

The `.gitignore` file is configured to exclude these files from version control, so they won't be committed to git. This is the correct behavior - these files should not be in version control.
